冒泡排序

核⼼：冒泡，持续⽐较相邻元素，⼤的挪到后⾯，因此⼤的会逐步往后挪，故称之为冒泡。

时间复杂度O(n^2)，空间复杂度O(1)，稳定，因为存在两两比较，不存在跳跃。

排序时间与输入无关，最好，最差，平均都是O(n^2)

选择排序

核⼼： 不断地选择剩余元素中的最⼩者。
1. 找到数组中最⼩元素并将其和数组第⼀个元素交换位置。
2. 在剩下的元素中找到最⼩元素并将其与数组第⼆个元素交换，直⾄整个数组排序。
时间复杂度O(n^2)， 空间复杂度O(1)
排序时间与输入无关，最佳情况，最坏情况都是如此, 不稳定

插⼊排序

核⼼：通过构建有序序列，对于未排序序列，在已排序序列中从后向前扫描(对于单向链表则只能从前往后遍历)，
找到相应位置并插⼊。 实现上通常使⽤in-place排序(需⽤到O(1)的额外空间)
1. 从第⼀个元素开始， 该元素可认为已排序
2. 取下⼀个元素， 对已排序数组从后往前扫描
3. 若从排序数组中取出的元素⼤于新元素， 则移⾄下⼀位置
4. 重复步骤3， 直⾄找到已排序元素⼩于或等于新元素的位置
5. 插⼊新元素⾄该位置
6. 重复2~5
时间复杂度O(n^2), 空间复杂度O(1)
排序时间与输入有关：输入的元素个数；元素已排序的程度。
最佳情况，输入数组是已经排好序的数组，运行时间是n的线性函数； 最坏情况，输入数组是逆序，运行时间是n的二次函数。

public static void insertionSort(int[] array) {
    int len = array.length;
    for (int i = 0; i < len; i++) {
        int index = i, array_i = array[i];
        while (index > 0 && array[index - 1] > array_i) {
            array[index] = array[index - 1];
            index -= 1;
        } 
        array[index] = array_i;
        /* print sort process */
        for (int item : array) {
            System.out.print(item + " ");
        } 
        System.out.println();
    }
}
